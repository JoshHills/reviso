JavaScript: Basics

Attention: This topic assumes some casual pre-existing knowledge of programming concepts. No dice? Go edumacate yourself!

You see that light on the horizon? You’ve finally made it to the end-of-the-line, and there’s no better topic in which to go out with a bang. Here, I’ll provide a brief run-down of the last building-block composing modern websites, which adds the last key step of functionality- interactivity. JavaScript Is a broad subject to cover, and can be looked at on a microcosmic scale if such a thing took your fancy. For the purposes of revision, I’ll touch on the most important relevent elements, such as how to include it in your website, what it does on a basic level, and then provide some catalysts for further reading. Are you excitedi'm excited!!?

JavaScript is the last piece of the puzzle.
It enables a greater depth of interaction for the users of your website.
The possibilities are endless, and as a result there's a lot to know.

JavaScript is a scripting language that executes in your Web Browser- it is embedded in your files, and has many favourable features for web-development; it’s dynamically typedthis makes it very relaxed, which is both a curse and a blessing as debugging can be hard to get to grips with, meaning you don’t have to specify what kinds of variables you are creating. It was introduced from a dualistic background of development by both Microsoft and Sun, and allows a user to interact with the DOM.

This is incredibly important, as it transforms your now pretty-looking, relatively static web-pages into applications. At a basic level, like other programming-languages, you can create variables and logical constructs, as well as functions; what you do inside these is literally limitless- you can register event-listeners to react to changes in the web-page, which may encompass anything from creating a game to processing form informatonquite effectively too, with tools such as ‘regex’.

Gimme!

Similar to CSS, including javascript in your website can be done by actually including the code in your document’s headto load before the dom loads or bodyto load after the body has loaded using <script> tags, however it’s better practice, especially with larger files, to link to them externally:

<script src="scripts/sections/javascript-1.js"></script>

You can see a bunch of intialisation scripts loaded into this page in the source!

As for interacting with your code, you can output to the console with console.log(string);, and actually run functions by namerunning using parentheses, and retrieiving the literal function-as-string by omitting them. If you want to take a look at the javascript model of your user-profile (and even tweak it, you cheater), you can type ‘user’ into the console and mess with the object it returns!

DOM Interaction:

Being able to navigate through the DOM in javascript is kindred to navigating your computer’s filesystem using the terminal, only slightly more confusing.

Without necessarily knowing the direct route to an element via the heirarchy, there are several functions you should memorise (they're all pretty self-explanatory):

document.getElementById()

document.getElementsByClassName()

document.getElementsByName()

document.getElementsByTagName()

The query selector lets you use CSS-style targeting to find and return elements:

document.querySelector()

A more comprehensive list can be found here!

Regular Expressions:

These provide a standardised method of pattern matching for text- they’re like search criteria to draw substrings from content in JavaScript- they are usful in validation. To construct a ‘regexp’, you concoct a string of meta-characters that have specific meanings, best outlined in the cheat-sheet here. Now you know of their existence, when you inevitable come across a reason to use them you should know of a means to solve your problem!

Graceful Degradation:

Having meandered over some quick basics of JavaScript, we should take that backseat to chat about good practices. The big issue with including script-functionality in a web-page is that it shouldn’t be dependent upon it if you still want to convey information alongside it- this is because, if it breaks or there are any compatibility issues from your users, it will not degrade helpfully! By scripting practicallyusing tools wisely, adopting other frameworks..., you should foster unobtrusive code that lends itself well to graceful degradation.

This is especially important when utilising pluginsother people's code-libraries that do useful jobs for you, as they aren't maintained by you- you never know when something could go wrong!
